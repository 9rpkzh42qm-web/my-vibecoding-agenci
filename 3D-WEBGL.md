# 3D-WEBGL.md

Этот файл использовать только если задача связана с:
- анимированным сайтом;
- сложной 3D-графикой;
- WebGL;
- Three.js;
- React Three Fiber / Drei;
- движущимися 3D-объектами;
- стеклом, металлом, частицами, свечением.

## Технический стек для 3D
- Библиотеки: Three.js как база.
- Если проект на React: React Three Fiber + Drei.
- Форматы моделей: GLTF / GLB.
- Рендеринг: WebGLRenderer.
- Для современных сцен использовать physicallyCorrectLights.
- Для сцен с 3D-моделями по возможности использовать оптимизированные ассеты и не дублировать тяжёлые ресурсы без необходимости.

## Материалы и отражения
- Для золота: MeshStandardMaterial с metalness: 1.0 и roughness: 0.1-0.2.
- Для стекла/сферы: MeshPhysicalMaterial с transmission: 1.0, roughness: 0.0, thickness: 1.0.
- Для реалистичных отражений обязательно использовать HDR / Environment Map.
- Следить, чтобы материалы выглядели дорого, а не пластиково.
- Если сцена перегружена, сначала упрощать геометрию, а не добавлять лишние эффекты.

## Движение и анимация
- Делать независимое вращение объектов по X, Y, Z с разной скоростью.
- Использовать requestAnimationFrame или штатный анимационный механизм фреймворка.
- Добавлять лёгкие микроколебания, чтобы движение выглядело живым.
- Не делать анимацию слишком резкой или однообразной.
- Если есть несколько колец или объектов, они должны вращаться не синхронно.

## Частицы и атмосфера
- Для парящих частиц использовать THREE.Points.
- Частицы должны иметь случайный разброс координат.
- Движение частиц должно быть медленным и плавным.
- Можно имитировать пыль, искры, сияние или атмосферные частицы.
- Частицы должны усиливать сцену, а не превращать её в шум.

## Свет и сцена
- Использовать минимум источников света, но достаточно для объёма.
- Не перегружать сцену десятками lights.
- Следить за объёмом, отражениями и глубиной.
- Если нужен красивый металлик или стекло, сначала настраивать свет, потом материалы.
- Для мобильных устройств важен баланс между красотой и производительностью.

## Производительность
- Не делать сцену слишком тяжёлой.
- Оптимизировать модели GLTF/GLB.
- Стараться переиспользовать ассеты.
- Избегать лишних вычислений в каждом кадре.
- Если сцена сложная, предлагать упрощённый fallback для слабых устройств.
- На телефоне сцена не должна лагать или ломать скролл.

## Как отвечать на такие задачи
- Сначала предложи композицию сцены.
- Потом опиши материалы и свет.
- Потом добавь анимацию и частицы.
- Потом проверь мобильную адаптацию.
- Если нужен код, делай его модульно и аккуратно.
- Если задача клиентская, можно сослаться на этот файл и собрать сцену по его правилам.

## Проверка качества
Перед финалом проверь:
- есть ли одна визуальная идея;
- есть ли глубина и объём;
- не перегружена ли сцена;
- достаточно ли она плавная;
- не забыты ли материалы, свет, частицы и отражения;
- подходит ли это для mobile-first.
