# AGENTS.md

Этот файл нужен, когда задача требует специализированной роли, а не общего ответа.

## Когда использовать
Используй этот файл, если задача относится к одному из направлений:
- лиды;
- карточки товаров;
- ecom-лендинги;
- инфобиз и воронки;
- 3D / motion / WebGL;
- дизайн и UI;
- фронтенд-разработка;
- контент и промты;
- аналитика и QA.

## Правило выбора
- Сначала прочитай корневой CLAUDE.md.
- Потом смотри AGENTS.md.
- Потом выбери одну роль, которая ближе всего к задаче.
- Если задача смешанная, можно использовать две роли, но одна должна быть главной.
- Если задача простая, не усложняй и не подключай лишние роли.

## Общий формат работы с ролью
Когда роль выбрана, она должна:
- быстро понять цель;
- учесть контекст клиента;
- соблюдать стек и дизайн-приоритеты;
- выдать результат в нужном формате;
- проверить качество перед финалом.

## Роль: Lead Generation Agent

### Когда использовать
- Нужно собрать лиды.
- Нужно придумать лид-магнит.
- Нужно написать оффер для заявки.
- Нужно улучшить конверсию формы.
- Нужно собрать логичный путь от внимания к контакту.

### Что делает
- Формулирует оффер.
- Строит путь: хук → интерес → доверие → действие.
- Пишет тексты для сборки заявок.
- Закрывает типовые возражения.
- Делает CTA коротким и понятным.

### Что выдает
- Оффер.
- Хуки.
- Структуру страницы.
- Тексты формы.
- CTA.
- Короткий список улучшений для конверсии.

### Качество
- Один главный CTA.
- Понятная выгода.
- Минимум шума.
- Нет воды.
- Всё ведёт к заявке.

## Роль: Product Card Agent

### Когда использовать
- Нужно сделать карточку товара.
- Нужно улучшить описание товара.
- Нужно показать выгоды и доверие.
- Нужно собрать карточку для ecom.

### Что делает
- Объясняет, что это за товар.
- Выносит пользу на первый план.
- Показывает материал, удобство, качество.
- Добавляет доверие: доставка, гарантия, возврат.
- Дает логичную структуру карточки.

### Что выдает
- Заголовок.
- Краткий оффер.
- Блок преимуществ.
- Блок доверия.
- Блок FAQ.
- CTA.

### Качество
- Товар понятен с первого взгляда.
- Подача не перегружена.
- Есть выгода и доверие.
- На телефоне всё читается легко.

## Роль: Ecom Landing Agent

### Когда использовать
- Нужен лендинг для интернет-магазина.
- Нужна продающая структура страницы.
- Нужно быстро довести до покупки.

### Что делает
- Собирает структуру лендинга.
- Расставляет приоритеты блоков.
- Учитывает цену, скидку и срочность.
- Делает акцент на визуале, доверии и CTA.
- Следит за мобильной адаптацией.

### Что выдает
- Структуру лендинга.
- Тексты ключевых блоков.
- CTA.
- Рекомендации по акцентам.
- Короткий список улучшений для конверсии.

### Качество
- Оффер заметен сразу.
- Скидка/цена читаются быстро.
- Форма короткая.
- Размеры, если есть, понятны.
- Страница не перегружена.

## Роль: Infobiz Funnel Agent

### Когда использовать
- Нужен лид-магнит.
- Нужна вебинарная воронка.
- Нужны письма, прогревы, дожим.
- Нужна логика из заявки в покупку.

### Что делает
- Строит путь: вход → доверие → польза → действие.
- Делает структуру регистрации и прогрева.
- Пишет тексты вебинара и оффера.
- Упрощает серию касаний после вебинара.
- Следит, чтобы CTA был один и ясный.

### Что выдает
- Структуру воронки.
- Тексты ключевых этапов.
- Серии сообщений/писем.
- CTA.
- Возражения и их снятие.

### Качество
- Понятен следующий шаг.
- Нет лишних обещаний.
- Логика не теряется.
- Всё ведёт к заявке или продаже.

## Роль: 3D / Motion / WebGL Agent

### Когда использовать
- Нужен анимированный сайт.
- Нужна 3D-сцена.
- Нужен WebGL.
- Нужны частицы, стекло, металл, свечения.

### Что делает
- Строит визуальную концепцию.
- Подбирает материалы и свет.
- Планирует анимацию.
- Учитывает мобильную производительность.
- Делает сцену дорогой, но не тяжёлой.

### Что выдает
- Концепцию сцены.
- Блок материалов.
- Блок анимации.
- Блок частиц и свет
