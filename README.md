# My Vibecoding Agency

Это корень проекта. Здесь лежат главные правила, структура и быстрый путь к работе.

## Что открыть сначала
1. CLAUDE.md в корне.
2. README.md в корне.
3. Нужную клиентскую папку.
4. CLAUDE.md нужного уровня.
5. 06-PROMPTS-BLOG.md, если нужен готовый промт.
6. 3D-WEBGL.md, если задача про анимированный сайт или 3D.

## Как работать каждый день
- Сначала пойми задачу.
- Потом выбери нужный уровень контекста.
- Потом проверь, не конфликтуют ли правила.
- Потом делай ответ, код или текст.
- В конце проверь логику, простоту и качество.

## Приоритет файлов
- Корневой CLAUDE.md — общие правила.
- Клиентский CLAUDE.md — правила конкретного клиента.
- Файл связки или лендинга — самый точный контекст.
- 3D-WEBGL.md — отдельные правила для WebGL и анимированных 3D-сцен.
- Если правила конфликтуют, важнее более глубокий файл.

## Для кода
- Используй только разрешённый стек.
- Стили пиши через Tailwind-классы.
- Не используй inline style.
- Делай mobile-first.
- Валидируй формы на клиенте.
- Для картинок используй loading="lazy".
- Следи за семантикой и доступностью.

## Для контента и промтов
- Если задача про контент, продажи, упаковку или дизайн, сначала смотри 06-PROMPTS-BLOG.md.
- Если нужен быстрый результат, бери готовый промт и подставляй свои данные.
- Если задача новая, сначала выбери роль, потом задай контекст, потом результат.

## Перед финалом
Проверь:
- логика;
- простота;
- адаптивность;
- доступность;
- один главный CTA;
- нет ли лишней воды.

## Главное правило
Держи верхний файл коротким.
Детали выноси в отдельные папки.
Не смешивай всё в один документ.
