## Когда использовать
Этот блок включай только если задача про:
- анимированный сайт;
- 3D-сцену;
- WebGL;
- вращающиеся объекты;
- стекло, металл, частицы;
- эффект дорогой motion-вёрстки.

## Технический стек для 3D
- Библиотеки: Three.js (база) + React Three Fiber / Drei, если проект на React.
- Форматы моделей: GLTF / GLB.
- Рендеринг: WebGLRenderer.
- Для современных сцен использовать physicallyCorrectLights.
- Для React-сцен по возможности переиспользовать ассеты и не дублировать тяжёлые модели.

## Настройки материалов и физики движений
- Для золота: MeshStandardMaterial с metalness: 1.0 и roughness: 0.1-0.2.
- Для стекла/сферы: MeshPhysicalMaterial с transmission: 1.0, roughness: 0.0, thickness: 1.0.
- Для отражений обязательно использовать HDR / Environment Map.
- Для динамичных сцен делать независимое вращение элементов по X, Y, Z с разной скоростью.
- Анимацию делать через requestAnimationFrame или подходящий механизм анимации в React.
- Не делать движение механическим: добавлять лёгкую разницу скоростей, плавность и микроколебания.

## Эффекты сцены
- Для парящих частиц использовать THREE.Points.
- Частицы должны иметь случайный разброс координат.
- Движение частиц должно быть медленным и плавным.
- Можно использовать эффект золотой пыли, искр или атмосферного свечения.
- Если сцена перегружена, сначала упрощай геометрию, а не добавляй лишние эффекты.

## Свет и визуал
- Использовать минимум источников света, но достаточно для объёма.
- Не перегружать сцену десятками lights.
- Следить за реалистичными отражениями на металле и стекле.
- Для красивого результата использовать правильные тени, но не в ущерб производительности.

## Производительность
- Не дублировать тяжёлые объекты без нужды.
- Использовать оптимизированные модели GLTF/GLB.
- Не делать сцену слишком тяжёлой для мобильных устройств.
- Если сцена сложная, предлагать более лёгкий fallback для слабых устройств.

## Стиль результата
- Сцена должна выглядеть дорого, плавно и современно.
- Анимация должна быть заметной, но не агрессивной.
- Важно ощущение глубины, света и объёма.
- На мобильных устройствах всё должно оставаться стабильным и читаемым.

## Проверка перед финалом
- Есть ли единая визуальная идея.
- Не перегружена ли сцена.
- Есть ли баланс между красотой и скоростью.
- Работает ли это на mobile-first.
- Не забыты ли материалы, свет, частицы и плавность движения.
